1️⃣ Đăng nhập hệ thống (Lấy Token)
Method: POST

URL: https://localhost:3443/api/v1/auth/login

Body (JSON):

JSON
{
    "username": "member1",
    "password": "member123"
}
👉 Mục đích: Copy chuỗi token trong kết quả trả về để dùng cho các bước sau.

2️⃣ Khảo sát kho hàng (Tìm sản phẩm)
Method: GET

URL: https://localhost:3443/api/v1/products?page=1&limit=5&search=Ryzen

Auth: Không cần.

👉 Mục đích: Lấy pro_id của sản phẩm muốn mua (Ví dụ ID: 2).

3️⃣ Đặt hàng (Chốt đơn)
Method: POST

URL: https://localhost:3443/api/v1/orders

Auth (Bearer Token): Dán token ở Bước 1 vào.

Body (JSON):

JSON
{
    "guest_name": "Anh Khách Hàng Member 1",
    "guest_phone": "0987654321",
    "shipping_address": "Quận 1, TP. Hồ Chí Minh",
    "product_id": 2, 
    "quantity": 1
}
👉 Lưu ý: Sau bước này, đơn hàng sẽ có trạng thái PENDING.

4️⃣ 🛠️ (Bước đệm Admin) Giao hàng thành công
Mở phần mềm Database (Navicat/Workbench) chạy lệnh SQL sau để hoàn thành đơn hàng:

SQL
UPDATE orders SET status = 'COMPLETED' WHERE status = 'PENDING';
5️⃣ Viết Đánh giá (Review)
Method: POST

URL: https://localhost:3443/api/v1/reviews

Auth (Bearer Token): Dán token ở Bước 1 vào.

Body (JSON):

JSON
{
    "productId": 2 || other,
    "rating": 1-5,
    "comment": "Chip mạnh, render video vèo vèo! "your comment" "
}
6️⃣ Xem danh sách Đánh giá
Method: GET

URL: https://localhost:3443/api/v1/reviews/2

Auth: Không cần.

👉 Mục đích: Lấy danh sách hiển thị ra giao diện Website giống như kết quả bạn vừa làm thành công.

1️⃣ Upload ảnh Sản phẩm
Method: POST

URL: https://localhost:3443/api/v1/products/1/images
(Thay 1 bằng ID sản phẩm thực tế).

Auth: Bearer Token (Yêu cầu quyền STAFF hoặc ADMIN).

Body (form-data): Key là image, chọn file ảnh (< 2MB).

👉 Mục đích: Lưu ảnh linh kiện lên server và lấy URL hiển thị ra giao diện.

2️⃣ Đăng nhập hệ thống (Auth)
Method: POST

URL: https://localhost:3443/api/v1/auth/login

Auth: Không cần.

Body (JSON): {"email": "...", "password": "..."}.

👉 Mục đích: Lấy Access Token để sử dụng cho các API yêu cầu quyền hạn phía sau.

3️⃣ Tra cứu bảo hành (Inventory)
Method: GET

URL: https://localhost:3443/api/v1/inventory/warranty/SER123456 tạm

Auth: Không cần.

👉 Mục đích: Cho phép khách hàng nhập số Serial để kiểm tra ngày hết hạn bảo hành của linh kiện.

4️⃣ Tạo đơn hàng mới (Order)
Method: POST

URL: https://localhost:3443/api/v1/orders

Auth: Bearer Token (Phải đăng nhập).

Body (JSON): Thông tin sản phẩm, số lượng, mã giảm giá (nếu có).

👉 Mục đích: Lưu đơn hàng vào cơ sở dữ liệu và trừ tồn kho thực tế.

5️⃣ Lấy danh sách Sản phẩm (Có bộ lọc)
Method: GET

URL: https://localhost:3443/api/v1/products?brand=ASUS&minPrice=1000000

Auth: Không cần.

👉 Mục đích: Hiển thị danh sách linh kiện trên trang chủ, hỗ trợ lọc theo hãng và khoảng giá.
